# BudCast Dark Premium Moody Design

Date: 2026-04-24
Status: Approved visual direction
Owner: Codex

## Decision

BudCast will use `Dark Premium Moody` as the default visual system across web and native.

`Editorial Operator` remains the structural discipline underneath the aesthetic system. In practice, this means BudCast should look cinematic, exclusive, and high-trust, while still behaving like serious marketplace software with clear hierarchy, readable task flow, and operational restraint.

This is the source-of-truth design decision for the next product pass.

## Intent

BudCast should feel like a protected premium marketplace where brands and creators handle money, reputation, proof, and collaboration in an environment that feels expensive and controlled.

The product should not feel like:
- generic black-glass SaaS
- gamer-neon dark mode
- soft lifestyle packaging with weak operational clarity
- card-heavy dashboard boilerplate

The product should feel like:
- cinematic and composed
- cannabis-adjacent without cliché
- credible for B2B brand operators
- desirable for creators
- structured enough that action priority is obvious

## Core Visual Thesis

BudCast is a midnight-market control surface.

The system should combine:
- dark lacquer surfaces
- smoked layering
- restrained bronze and olive accents
- selective editorial serif moments
- slow, intentional motion
- highly legible operational content

The mood is premium and atmospheric, but the product behavior is disciplined and trustworthy.

## Design Principles

### 1. Mood Leads, Discipline Governs

Dark Premium Moody is the aesthetic lead.

Editorial Operator is the behavioral guardrail.

Every surface should ask:
- does this feel expensive and protected?
- can the user still tell what matters first?

If the mood hurts clarity, clarity wins.
If the structure feels generic, mood should be strengthened through composition rather than more UI clutter.

### 2. Premium Through Restraint

Premium should come from:
- contrast
- spacing
- composition
- hierarchy
- selective highlight moments

Premium should not come from:
- too many glows
- too many accent colors
- excessive frosted panels
- ornamental motion
- decorative icon overload

### 3. Marketplace Trust Is The Product

The visual system must reinforce:
- proof review
- creator credibility
- payout visibility
- brand confidence
- operational seriousness

Design should make trust-heavy moments feel more substantial, not more theatrical.

### 4. Cannabis-Adjacent, Not Cannabis-Literal

Category cues should come from tone and materiality:
- olive undertones
- brass warmth
- smoked neutrals
- velvet-room contrast

Avoid:
- leaf overload
- bright green weed stereotypes
- dispensary cliché language
- novelty cannabis visual tropes

## Color System

### Base

- Primary background: near-black with slight olive/brown warmth
- Secondary background: smoked charcoal
- Elevated surface: lacquered dark graphite with subtle warmth
- Soft contrast panel: muted asphalt / dark mineral

### Accents

- Metallic warm accent: bronze / muted brass
- Botanical accent: restrained olive green
- Trust accent: desaturated emerald for verified or success states only

### Rules

- Green appears rarely and must mean something
- Warm metallic tones are for guidance and premium emphasis, not constant decoration
- White should be softened slightly, not blue-white
- Contrast must remain accessible on all primary tasks and status views

## Typography

### Role Split

- Serif: reserved for hero headlines, major section statements, and a few proof-heavy emphasis moments
- Sans: all operational copy, tables, labels, controls, forms, and status text

### Rules

- Serif never drives dense workflow reading
- Working surfaces must scan quickly without decorative friction
- Heading rhythm should feel editorial, but utility text should feel sober and crisp

## Surface Language

### Use

- Large planes instead of repeated small cards
- Strong frame boundaries between command area, evidence area, and ledger/list area
- Smoked and dark-glass treatments only when they improve grouping and focus

### Avoid

- KPI mosaics
- repeated pill clutter
- too many nested panels
- over-rounded “soft luxury” blocks that reduce seriousness

## Layout Model

### Web

Default workspace structure:
- clear command masthead
- one primary action zone
- one supporting evidence rail
- one operational ledger or queue

The web app should feel like an operator desk, not a showcase page.

### Native

The native app should share the dark system, but be slightly more approachable and kinetic than brand web.

Default creator structure:
- high-clarity hero context
- focused discovery feed
- visible application/submission status
- calm trust cues around proof and payout

Native should feel premium and personal, not severe.

## Motion

### Desired Motion

- slow reveal
- intentional hover lift
- route transitions with presence
- subtle signal movement for live marketplace energy
- ceremonial emphasis around proof, verification, and payout states

### Avoid

- bounce
- noisy looping animation
- attention-seeking shimmer on routine surfaces
- motion that feels like decoration rather than information

## Product Translation

### Brand Web Workspace

Should feel:
- commanding
- exclusive
- focused
- high-confidence

Needs:
- stronger dark structural shell
- fewer equal-weight content blocks
- more obvious task priority
- premium review and payout moments

### Creator Mobile App

Should feel:
- aspirational
- premium
- alive
- trustworthy

Needs:
- dark mood with slightly more softness than web
- stronger campaign discovery storytelling
- clear status progression for applications and submissions
- payment and proof moments that feel substantial and transparent

### Top-Of-Funnel Web

Should feel:
- cinematic
- memorable
- premium
- marketplace-specific

Needs:
- stronger poster-like composition
- fewer generic content boxes
- a more obvious sense that BudCast is a protected premium exchange

## Guardrails

Do not allow the system to drift into:
- generic dark SaaS
- fintech clone aesthetics
- cyberpunk / neon weed aesthetics
- luxury-for-luxury’s-sake softness
- unreadable low-contrast copy

BudCast should remain:
- operational
- premium
- category-aware
- differentiated

## Acceptance Criteria

The design pass is successful when:
- the web app immediately feels darker, more premium, and more controlled
- the product still reads clearly in under a few seconds
- brands feel the workspace is worthy of serious campaign spend
- creators feel the mobile product is premium rather than generic
- proof, trust, and payout states feel more emotionally weighted
- the visual system is consistent across web and native

## Scope Of Next Implementation Pass

Apply the approved system in this order:

1. Web brand shell and dashboard
2. Web top-of-funnel surfaces
3. Web operational sub-pages: submissions, applicants, campaign detail, campaign builder
4. Native creator home and discovery surfaces
5. Native profile, submissions, and applications
6. Shared tokens and interaction polish pass

## Open Decisions

These are intentionally still open for later product tuning, not blockers for implementation:

- how dark the creator app should be relative to the brand workspace
- whether premium campaign/program areas should become even darker or more ceremonial than the core app
- whether a lighter public-marketing layer is ever needed later for acquisition

## Recommendation

Proceed with `Dark Premium Moody` as the default BudCast system and use `Editorial Operator` as the structural review lens during implementation.

When choosing between beauty and clarity, preserve clarity.
When choosing between generic clarity and distinctiveness, preserve distinctiveness through composition, not extra chrome.
